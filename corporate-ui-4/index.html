Corporate UI 4.x

Corporate UI is in the final stages to release a major update, version 4.x. With it comes several improvements based on feedback and insights from the previous versions of Corporate UI. The goal to deliver a consistent styling combined with reusable components remains the same, but the way of achieving that goal has received a few changes.

Web components

The most prominent change is to the technology used to create Web components. In this version of Corporate UI it has been decided to move away from Google Polymer to StencilJS. The reason for this change is the request for a more simple, lightweight and future proof Corporate UI by the developer community. Stencil generates standard web component that works in any framework or no framework at all. The logic is now moved from the runtime to the compilation phase to optimize load time. With this technology it is now possible to import component as a module. In addition, web components now available as self-contained components where the styling is scoped within the components. More information about Web components can be found in the new Corporate UI 4 repository.

Styling

Another change that was made is the configuration for styling to deliver a better developer experience. The style theme has now been separated from Corporate UI in order to give developers better control of the theming. Separating the style theme from Corporate UI enables parallel updates to be made to the different repository’s. Furthermore it enables for additional brand themes to be created and added to apply on the Corporate UI components. Applying the themes is made easy by using a new component called c-theme. All assets such as favicons, fonts, logotype and wordmark is now also bundled together and available in the theme package. While Scania will stick to the theme approved and aligned with Scania Identity Guidelines, this welcome suggestions how to expand the current theme to areas of use that has special requirements. Check out Scania theme repository for a more detail documentation.

Distribution

In addition to the CDN delivery, it is now possible to use Corporate UI and Scania theme as a package as it is available in NPM. This will enable projects to host assets from Corporate UI how they want without relying on an internet connection. Furthermore it will add flexibility to how they want to manipulate the package to fit their project’s needs.

CDN

Add link to the script by adding the following to the <head>. Make sure to include scania-theme script BEFORE the corporate-ui script.

<script src="https://static.scania.com/build/global/themes/scania/1.x/scania-theme.js"></script>
<script src="https://static.scania.com/build/global/4.x/corporate-ui.js"></script>

Replace x with available releases.
