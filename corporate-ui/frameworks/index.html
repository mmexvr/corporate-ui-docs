Corporate UI & Front End Apps

Using Corporate UI with Angular, React or an equivalent framework

Using the Corporate UI library with standard HTML pages is a very straight forward procedure. Although, when using it in a JS intense application based on any of the modern JS frameworks, you'll need to consider a few things.

What's happening in corporate-ui.js?

When calling the corporate-ui.js file a lot of things happens. The latest versions of the Scania fonts, logotype, favicons and the basic CSS and HTML framework (Scania Bootstrap) is loaded as well as the Polymer library. We also load the basic Web Components used by almost all applications; the Corporate Header, the Main Navigation, the Main Content element and the Corporate Footer. This doesn't work very well when an application is about to be initiated further on.

Wait for the application!

All you have to do to make the Corporate UI work in your application is to avoid loading the basic components right away and instead include them after your application is completely initiated. You simply load the Corporate UI with the ?preload=false parameter as you would load it in any HTML page header.

<script src="https://static.scania.com/build/global/3.5.3/js/corporate-ui.js?preload=false"></script>

Then, after your app has started, you can easily include the basic components by using the window.CorporateUi.baseComponents(); method. This will then load the components listed above and you're now ready to use those custom elements from wherever you need to apply them. In Angular, you could do it this way (or attached it to any other late event):

export class AppComponent { constructor(){ window.CorporateUi.baseComponents(); } }
